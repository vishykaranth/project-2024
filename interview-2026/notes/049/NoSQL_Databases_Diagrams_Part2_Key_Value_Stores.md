```
1. Provisioned Capacity:
   Read Capacity Units (RCU):
   - 1 RCU = 1 strongly consistent read/sec
   - 1 RCU = 2 eventually consistent reads/sec
   
   Write Capacity Units (WCU):
   - 1 WCU = 1 write/sec (up to 1KB)

2. On-Demand Mode:
   - Auto-scaling
   - Pay per request
   - No capacity planning

3. Batch Operations:
   BatchGetItem: Up to 100 items
   BatchWriteItem: Up to 25 items
   (Reduces round trips)

4. Query vs Scan:
   Query: Uses index (fast, efficient)
   Scan: Full table scan (slow, expensive)
```

## Key Concepts Summary

### Redis
```
- In-memory key-value store
- Rich data structures
- Pub/Sub messaging
- Streams for message queues
- Clustering for scalability
- Persistence (RDB, AOF)
```

### DynamoDB
```
- Managed NoSQL database
- Serverless, auto-scaling
- Global tables for multi-region
- Streams for change capture
- GSIs for flexible queries
- On-demand or provisioned capacity
```

### Caching Strategies
```
- Cache-Aside: Lazy loading
- Write-Through: Write to cache and DB
- Write-Back: Write to cache, async to DB
- TTL-based expiration
- Event-based invalidation
```
