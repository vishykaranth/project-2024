```
Modeling:
- Model relationships explicitly
- Use appropriate relationship types
- Add properties to relationships when needed
- Use labels effectively for node types
- Avoid deep hierarchies (use relationships)

Query Optimization:
- Use indexes on frequently queried properties
- Limit path depth in variable-length queries
- Use WHERE clauses early to filter
- Use PROFILE to analyze query performance
- Avoid cartesian products

Performance:
- Use appropriate relationship directions
- Batch operations when possible
- Use parameters in queries (not string concatenation)
- Monitor query execution times
- Use constraints and indexes wisely
```
