Schema Design:
- Use tags for indexed metadata (low cardinality)
- Use fields for actual measurements
- Avoid high-cardinality tags
- Use appropriate timestamp precision
- Design for query patterns

Write Optimization:
- Batch writes (1000+ points per batch)
- Use appropriate write consistency
- Monitor write throughput
- Handle backpressure gracefully
- Use compression when possible

Query Optimization:
- Use time-based WHERE clauses
- Leverage downsampled data for historical queries
- Use continuous queries for common aggregations
- Limit result sets
- Use appropriate time intervals

Retention and Storage:
- Implement retention policies
- Use downsampling for long-term storage
- Compress old data
- Monitor disk usage
- Plan for data growth

## Key Concepts Summary

### InfluxDB
```
- Purpose-built for time-series
- Line protocol for ingestion
- InfluxQL and Flux query languages
- TSM storage engine
- Retention policies
- Continuous queries
```

### TimescaleDB
```
- PostgreSQL extension
- SQL interface
- Automatic chunking
- Compression
- Continuous aggregates
- Full PostgreSQL compatibility
```

### Time-Series Patterns
```
- High write throughput
- Time-based queries
- Aggregations over time windows
- Downsampling for retention
- Compression for storage
- Continuous aggregations
```
