## Key Concepts Summary

### Cassandra
```
- Distributed, peer-to-peer architecture
- No single point of failure
- Tunable consistency
- CQL query language
- Wide-column data model
- Replication and sharding
```

### HBase
```
- Master-slave architecture
- Built on HDFS
- Strong consistency
- Java API, Thrift, REST
- Column families
- Automatic region splitting
```

### Wide-Column Design
```
- Partition key for distribution
- Clustering key for sorting
- Denormalization
- Time-series patterns
- Sparse data handling
```
