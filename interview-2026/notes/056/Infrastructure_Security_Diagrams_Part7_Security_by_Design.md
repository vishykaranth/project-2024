## Key Concepts Summary

### Security by Design
```
Early Integration: Security from start
Lower Cost: Fix early, not late
Better Security: Built-in, not bolted-on
Compliance: Ready from day one
```

### SDL Phases
```
Training: Security awareness
Requirements: Security requirements
Design: Threat modeling
Implementation: Secure coding
Verification: Security testing
Release: Security review
Response: Incident handling
```

### Threat Modeling
```
STRIDE: Threat categories
Assets: What to protect
Architecture: System design
Mitigation: Security controls
```

### Secure Coding
```
Input Validation: All input
Output Encoding: Context-aware
Authentication: Strong auth
Authorization: Every access
Error Handling: Secure errors
Cryptography: Strong crypto
```
