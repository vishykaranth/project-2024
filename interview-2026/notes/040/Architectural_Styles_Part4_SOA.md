```
Service-Oriented Architecture (SOA) is an architectural pattern
where applications are built as a collection of services that
communicate over a network.

Key Principles:
1. Service Reusability: Services can be reused
2. Service Autonomy: Services are independent
3. Service Statelessness: Services don't maintain state
4. Service Discoverability: Services can be discovered
5. Service Composability: Services can be composed
6. Standardized Service Contract: Well-defined interfaces
```

```
   - Quality of Service (QoS)

   - SLA requirements

   - Service owner
   - Service documentation
   - Service dependencies
```

```
- Independent data models
- Direct communication (REST/gRPC)
- Lightweight protocols
- Decentralized governance
- API contracts (OpenAPI)
- Business capability focus

When to Use SOA:
âœ… Large enterprise systems
âœ… Need for service reuse
âœ… Complex integration requirements
âœ… Legacy system integration

When to Use Microservices:
âœ… Modern applications
âœ… Independent deployment needs
âœ… Team autonomy required
âœ… Cloud-native applications
```
