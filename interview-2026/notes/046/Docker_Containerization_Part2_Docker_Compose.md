## Key Concepts Summary

### Docker Compose
```
- Orchestrates multi-container applications
- Single file (docker-compose.yml) defines entire stack
- Handles networking, volumes, dependencies
- Development and testing tool
```

### Networking
```
- Default network: All services can communicate
- Custom networks: Isolate services
- Service discovery: DNS-based using service names
- Port mapping: Expose services to host
```

### Service Communication
```
- Use service names as hostnames
- Internal ports for inter-service communication
- External ports only for host access
- Environment variables for configuration
```

### Volumes
```
- Named volumes: Persistent, managed storage
- Bind mounts: Direct host mapping
- Anonymous volumes: Temporary storage
- Data persistence across container restarts
```
