## Key Concepts Summary

### Image
```
- Immutable template
- Composed of read-only layers
- Stored in registry or locally
- Can be versioned with tags
- Shared layers save disk space
```

### Container
```
- Running instance of an image
- Has read-write layer on top of image
- Isolated from host and other containers
- Ephemeral (changes lost when removed)
- Can be started, stopped, restarted
```

### Dockerfile
```
- Text file with build instructions
- Each instruction creates a layer
- Order matters for caching
- Best practices: minimize layers, use .dockerignore
```

### Build Process
```
- Context sent to daemon
- Instructions executed sequentially
- Layers cached for faster rebuilds
- Final image tagged and stored
```
