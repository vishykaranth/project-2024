## Key Concepts Summary

### Volume Types
```
1. Named Volumes:
   - Managed by Docker
   - Persistent across container lifecycle
   - Best for production data

2. Bind Mounts:
   - Direct host mapping
   - Good for development
   - Host permissions apply

3. Anonymous Volumes:
   - Auto-generated names
   - Temporary storage
   - Removed with container

4. tmpfs:
   - In-memory storage
   - Very fast
   - Lost on container stop
```

### Volume Management
```
- Create: docker volume create
- List: docker volume ls
- Inspect: docker volume inspect
- Remove: docker volume rm
- Prune: docker volume prune
- Backup: tar from volume
- Restore: tar to volume
```
