---

## 1. Core Architecture Patterns

Depending on your business needs, I can design for various paradigms:

* **Serverless First:** Utilizing AWS Lambda/Azure Functions to minimize operational overhead and scale automatically with demand.
* **Containerized (K8s):** Designing robust Kubernetes clusters for complex, stateful applications that require fine-grained control over the runtime environment.
* **Event-Driven:** Using message brokers like Kafka or RabbitMQ to decouple services, ensuring that a spike in one area (e.g., "Orders") doesn't crash another (e.g., "Inventory").

---

## 2. The Cloud Design Checklist

When I design a platform for you, I walk through these critical layers:

| Layer | Component | Key Decision (ADR) |
| --- | --- | --- |
| **Compute** | EKS, Lambda, EC2 | Choose based on execution duration and state requirements. |
| **Networking** | VPC, Subnets, API Gateway | Implement **Zero Trust** and isolate public vs. private traffic. |
| **Data** | RDS, DynamoDB, S3 | Select based on the SQL vs. NoSQL decision tree we discussed. |
| **Security** | IAM, KMS, WAF | Apply the **Principle of Least Privilege** (PoLP). |
| **Observability** | CloudWatch, ELK, Prometheus | Define SLIs/SLOs to monitor system health in real-time. |

---

## 3. Deployment & Governance

A cloud platform is only as good as its delivery pipeline. I can help you design:

* **Infrastructure as Code (IaC):** Writing **Terraform** or **CloudFormation** templates to ensure your environments are reproducible and version-controlled.
* **Multi-Region Disaster Recovery:** Designing "Active-Passive" or "Active-Active" setups to ensure 99.99% availability even if an entire AWS Region goes dark.
* **Cost Management:** Implementing auto-scaling groups and Spot Instances to ensure you aren't paying for "idle" CPU time.
