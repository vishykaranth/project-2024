# Interview Question Answered | Daily Routine | Typical Day Look Like for Developers and QA

## Overview

Understanding the daily routine of developers and QA engineers helps set expectations and prepare for interviews. This covers typical day-to-day activities.

## Developer Daily Routine

```
┌─────────────────────────────────────────────────────────┐
│         Typical Developer Day                           │
└─────────────────────────────────────────────────────────┘

Morning (9-12):
├─► Standup meeting (15 min)
├─► Check emails/Slack
├─► Review code from yesterday
├─► Start new tasks
└─► Code development

Afternoon (12-5):
├─► Lunch break
├─► Continue development
├─► Code reviews
├─► Testing
├─► Documentation
└─► Team collaboration
```

## QA Engineer Daily Routine

```
┌─────────────────────────────────────────────────────────┐
│         Typical QA Day                                  │
└─────────────────────────────────────────────────────────┘

Morning:
├─► Standup meeting
├─► Test planning
├─► Test case execution
└─► Bug reporting

Afternoon:
├─► Regression testing
├─► Automation script development
├─► Bug verification
└─► Test reporting
```

## Key Activities

### Developers
- Writing code
- Code reviews
- Debugging
- Testing
- Documentation
- Meetings

### QA Engineers
- Test planning
- Test execution
- Bug reporting
- Automation
- Test reporting
- Collaboration

## Summary

Daily Routine:
- **Developers**: Code, review, test, collaborate
- **QA Engineers**: Plan, test, report, automate
- **Common**: Standups, meetings, collaboration
- **Focus**: Deliver quality software
