### Class Loading
```
1. ClassLoader Hierarchy
   - Bootstrap â†’ Extension â†’ Application â†’ Custom
   - Delegation model (parent first)

2. Loading Process
   - Loading: Find and read .class file
   - Linking: Verify, prepare, resolve
   - Initialization: Execute <clinit>()

3. Custom ClassLoader
   - Override findClass()
   - Custom loading logic
   - Use cases: plugins, hot reload, isolation

4. Dynamic Loading
   - Load classes at runtime
   - Plugin systems
   - Module systems (OSGi, JPMS)
```
