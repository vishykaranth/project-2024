## Quick Reference: When to Use

### Creational
- **Singleton**: Single instance needed
- **Factory**: Object creation complexity
- **Abstract Factory**: Families of objects
- **Builder**: Complex object construction
- **Prototype**: Cloning expensive objects

### Structural
- **Adapter**: Incompatible interfaces
- **Bridge**: Separate abstraction/implementation
- **Composite**: Tree structures
- **Decorator**: Add behavior dynamically
- **Facade**: Simplify complex subsystem
- **Flyweight**: Many similar objects
- **Proxy**: Control access to object

### Behavioral
- **Chain**: Multiple handlers for request
- **Command**: Encapsulate requests
- **Interpreter**: Language/grammar
- **Iterator**: Traverse collections
- **Mediator**: Reduce coupling
- **Memento**: Save/restore state
- **Observer**: One-to-many dependency
- **State**: Object behavior changes
- **Strategy**: Algorithms interchangeable
- **Template**: Algorithm skeleton
- **Visitor**: Operations on object structure
