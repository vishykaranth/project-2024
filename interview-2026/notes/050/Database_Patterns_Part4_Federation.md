    - Route orders query to DB2
    - Plan join execution

```

â”‚              Best Practices                                 â”‚
â””â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”˜

1. Minimize Cross-Database Queries
   - Design to avoid joins
   - Denormalize when needed
   - Use materialized views

2. Optimize Query Routing
   - Route to correct database
   - Parallel execution
   - Result caching

3. Handle Failures Gracefully
   - Circuit breakers
   - Retries with backoff
   - Fallback strategies

4. Monitor Performance
   - Query latency
   - Cross-DB query frequency
   - Database health

5. Transaction Management
   - Avoid distributed transactions
   - Use eventual consistency
   - Saga pattern when needed

6. Data Consistency
   - Eventual consistency model
   - Conflict resolution
   - Versioning

7. Security
   - Encrypt connections
   - Access control
   - Audit logging
```
