# Leadership & Management Answers - Part 26: Prioritization (Questions 126-130)

## Question 126: How do you prioritize technical work vs feature work?

### Answer

### Prioritization Framework

#### 1. **70-20-10 Rule**

```
┌─────────────────────────────────────────────────────────┐
│         Work Allocation                               │
└─────────────────────────────────────────────────────────┘

Feature Work (70%):
├─ New features
├─ Enhancements
├─ Bug fixes
└─ Customer requests

Technical Work (20%):
├─ Refactoring
├─ Technical debt
├─ Infrastructure
└─ Performance optimization

Innovation (10%):
├─ Proof of concepts
├─ Technology exploration
├─ Process improvements
└─ Learning time
```

## Question 127: What's your approach to prioritizing technical debt?

### Answer

### Technical Debt Prioritization

#### 1. **Priority Matrix**

```
┌─────────────────────────────────────────────────────────┐
│         Technical Debt Priority Matrix                │
└─────────────────────────────────────────────────────────┘

High Impact, High Effort:
├─ Major refactoring
└─ Prioritize strategically

High Impact, Low Effort:
├─ Quick wins
└─ Prioritize immediately

Low Impact, High Effort:
├─ Large but low value
└─ Defer or avoid

Low Impact, Low Effort:
├─ Small improvements
└─ Do when time permits
```

## Question 130: How do you handle competing priorities?

### Answer

### Competing Priorities Management

#### 1. **Resolution Process**

```
┌─────────────────────────────────────────────────────────┐
│         Competing Priorities Resolution                │
└─────────────────────────────────────────────────────────┘

1. Assess Impact
   ├─ Business value
   ├─ Urgency
   ├─ Dependencies
   └─ Resource requirements

2. Stakeholder Discussion
   ├─ Present options
   ├─ Discuss trade-offs
   └─ Reach agreement

3. Make Decision
   ├─ Document rationale
   ├─ Communicate clearly
   └─ Set expectations
```
