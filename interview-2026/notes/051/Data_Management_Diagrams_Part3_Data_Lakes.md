## Key Concepts Summary

### Data Lake
```
- Store all data types (structured, semi-structured, unstructured)
- Schema-on-read (apply schema when reading)
- Cost-effective storage
- Scalable architecture
- Supports multiple analytics tools
```

### Schema-on-Read
```
- Define schema when querying
- Store data as-is
- Flexible and adaptable
- Good for unknown or changing schemas
- Enables exploratory analytics
```

### Data Lake Zones
```
1. Raw Zone: Original data
2. Cleaned Zone: Validated data
3. Curated Zone: Business-ready data
4. Archive Zone: Historical data
```

### Best Practices
```
1. Organize data in zones
2. Use columnar formats (Parquet, ORC)
3. Implement data governance
4. Catalog and metadata management
5. Security and access control
6. Lifecycle management
```
