# Part 45: AWS Cloud Fundamentals - Quick Revision

## Compute Services

- **EC2**: Virtual servers, scalable compute capacity
- **ECS/EKS**: Container orchestration, ECS (AWS managed), EKS (Kubernetes)
- **Lambda**: Serverless functions, event-driven, pay per execution
- **Elastic Beanstalk**: Platform as a service, easy deployment

## Storage Services

- **S3**: Object storage, scalable, 99.999999999% durability
- **EBS**: Block storage for EC2, persistent volumes
- **EFS**: Managed NFS, shared file system
- **Glacier**: Archive storage, low cost, retrieval time

## Database Services

- **RDS**: Managed relational databases (PostgreSQL, MySQL, etc.)
- **DynamoDB**: NoSQL database, serverless, auto-scaling
- **ElastiCache**: In-memory caching (Redis, Memcached)
- **Redshift**: Data warehouse, analytics

## Networking

- **VPC**: Virtual private cloud, isolated network
- **Load Balancer**: ALB (application), NLB (network), CLB (classic)
- **CloudFront**: CDN, global content delivery
- **Route 53**: DNS service, domain management

## Key Concepts

- **Regions & Availability Zones**: Geographic distribution, high availability
- **IAM**: Identity and access management, security
- **Auto Scaling**: Automatically adjust capacity based on demand
- **CloudWatch**: Monitoring, logging, alerting
