📚 Weeks 1-2: Architectural Foundations & Scale

Distributed Systems Fundamentals

6-8 hours
  • CAP theorem deep dive and practical implications
  • Consistency models: strong, eventual, causal
  • Consensus algorithms: Paxos, Raft, Byzantine fault tolerance
  • Time and ordering in distributed systems (Lamport clocks, vector clocks)

Scalability Patterns & Techniques

5-7 hours
  • Horizontal vs vertical scaling trade-offs
  • Sharding strategies: hash-based, range-based, geography-based
  • Load balancing algorithms and consistent hashing
  • Replication strategies and eventual consistency patterns

Data Storage Architecture

6-8 hours
  • SQL vs NoSQL: when to use each
  • Database internals: B-trees, LSM trees, indexes
  • Data modeling for scale: denormalization, CQRS
  • Data warehousing vs data lakes vs lakehouses

Practical Exercise: Design Twitter/X at Scale

4-5 hours
Design end-to-end architecture covering: tweet ingestion, timeline generation, search, notifications, and analytics

⚡ Weeks 3-4: Performance Engineering & Reliability

Caching Strategies

5-6 hours
  • Cache hierarchies: L1/L2/CDN/application/database
  • Eviction policies: LRU, LFU, ARC
  • Cache invalidation strategies and consistency
  • Write-through vs write-back vs write-around

High Availability & Fault Tolerance

6-8 hours
  • SLA/SLO/SLI design and error budgets
  • Circuit breakers, bulkheads, and retry patterns
  • Health checks and graceful degradation
  • Multi-region active-active architectures

Performance Optimization

5-7 hours
  • Profiling and bottleneck identification
  • Database query optimization and indexing strategies
  • API design for performance: pagination, filtering, batch operations
  • Asynchronous processing and message queues

Practical Exercise: Design Netflix Video Streaming

4-5 hours
Focus on CDN strategy, adaptive bitrate streaming, regional failover, and handling millions of concurrent streams

📊 Weeks 5-6: Data Processing & Analytics

Stream Processing Architecture

6-7 hours
  • Event-driven architectures and event sourcing
  • Kafka, Kinesis, Pub/Sub comparison
  • Stream processing frameworks: Flink, Spark Streaming
  • Exactly-once semantics and idempotency

Batch Processing & ETL

5-6 hours
  • MapReduce paradigm and modern alternatives
  • Data pipeline orchestration: Airflow, Prefect
  • Delta Lake, Iceberg for ACID transactions
  • Incremental processing and change data capture

Real-time Analytics

5-6 hours
  • OLAP vs OLTP database design
  • Columnar databases: Clickhouse, Druid
  • Time-series databases and aggregation strategies
  • Approximate algorithms: HyperLogLog, Count-Min Sketch

Practical Exercise: Design Uber's Real-time Pricing & Analytics

4-5 hours
Design surge pricing system, driver-rider matching, real-time dashboards for millions of concurrent users

API Design & Versioning

Service Discovery & Configuration

4-5 hours

Practical Exercise: Design Shopify/Amazon Checkout System

5-6 hours
Multi-service architecture for inventory, payments, order processing, notifications with strong consistency requirements
