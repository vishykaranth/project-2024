## Introduction

**Consistent Hashing** is a distributed hashing technique used in distributed systems to minimize the number of keys that need to be remapped when hash table slots (nodes/servers) are added or removed. It was first introduced in 1997 by David Karger et al. at MIT.

### Why Consistent Hashing?

In distributed systems, we often need to:
- **Distribute data** across multiple servers/nodes
- **Load balance** requests across servers
- **Scale horizontally** by adding/removing servers
- **Minimize data movement** when servers join or leave

Traditional hashing fails when the number of servers changes, causing massive data rehashing. Consistent hashing solves this problem elegantly.

## The Problem with Traditional Hashing

### Traditional Hash Function

**Simple Modulo Hashing**:
```python
def hash_function(key, num_servers):
    return hash(key) % num_servers
```

**Example**:
```python
# 3 servers
servers = ['Server1', 'Server2', 'Server3']
num_servers = 3

# Data distribution
data = {
    'user:1': hash('user:1') % 3 = 0 → Server1
    'user:2': hash('user:2') % 3 = 1 → Server2
    'user:3': hash('user:3') % 3 = 2 → Server3
    'user:4': hash('user:4') % 3 = 0 → Server1
    'user:5': hash('user:5') % 3 = 1 → Server2
}
```

### The Problem: Server Addition/Removal

**Scenario 1: Adding a Server**

```
Before: 3 servers
After: 4 servers

Old mapping (mod 3):
user:1 → Server1 (hash % 3 = 0)
user:2 → Server2 (hash % 3 = 1)
user:3 → Server3 (hash % 3 = 2)
user:4 → Server1 (hash % 3 = 0)
user:5 → Server2 (hash % 3 = 1)

New mapping (mod 4):
user:1 → Server1 (hash % 4 = 1) ❌ MOVED
user:2 → Server2 (hash % 4 = 2) ❌ MOVED
user:3 → Server3 (hash % 4 = 3) ❌ MOVED
user:4 → Server4 (hash % 4 = 0) ❌ MOVED
user:5 → Server1 (hash % 4 = 1) ❌ MOVED
```

**Result**: **100% of keys need to be remapped!**

**Scenario 2: Removing a Server**

```
Before: 3 servers
After: 2 servers

Old mapping (mod 3):
user:1 → Server1
user:2 → Server2
user:3 → Server3
user:4 → Server1
user:5 → Server2

New mapping (mod 2):
user:1 → Server1 (hash % 2 = 0) ❌ MOVED
user:2 → Server2 (hash % 2 = 1) ❌ MOVED
user:3 → Server1 (hash % 2 = 0) ❌ MOVED
user:4 → Server2 (hash % 2 = 1) ❌ MOVED
user:5 → Server1 (hash % 2 = 0) ❌ MOVED
```

**Result**: **100% of keys need to be remapped!**

### Problems with Traditional Hashing

1. **Massive Rehashing**: Adding/removing servers causes all keys to be remapped
2. **High Network Traffic**: Moving all data between servers
3. **Cache Invalidation**: All caches become invalid
4. **Service Disruption**: During rehashing, data may be unavailable
5. **Load Imbalance**: Uneven distribution of keys

## What is Consistent Hashing?

**Consistent Hashing** is a special kind of hashing that:
- Maps both **keys** and **servers** to the same hash space (usually a circle/ring)
- Uses a **hash ring** where both keys and servers are placed
- Assigns each key to the **first server** encountered when moving clockwise
- **Minimizes remapping** when servers are added or removed

### Key Insight

Instead of using `hash(key) % num_servers`, consistent hashing:
1. Creates a **hash ring** (circular hash space)
2. Places **servers** on the ring using `hash(server_id)`
3. Places **keys** on the ring using `hash(key)`
4. Assigns key to the **nearest server clockwise**

---

## How Consistent Hashing Works

### Step 1: Create Hash Ring

**Hash Ring**: A circular space from 0 to 2^32 - 1 (or 2^64 - 1)

```
        0
        |
  2^32-1|0
        |
    Ring (circular)
        |
    2^16
```

**Visual Representation**:
```
                   0
                   |
        2^24       |       2^8
           \       |       /
            \      |      /
             \     |     /
              \    |    /
               \   |   /
                \  |  /
                 \ | /
                  \|/
                2^16
```

### Step 2: Place Servers on Ring

**Hash each server** and place it on the ring:

```python
def hash_server(server_id):
    return hash(server_id) % (2**32)

# Example
servers = ['Server1', 'Server2', 'Server3']
server_positions = {
    'Server1': hash('Server1') % (2**32) = 1000
    'Server2': hash('Server2') % (2**32) = 5000
    'Server3': hash('Server3') % (2**32) = 9000
}
```

**Visual**:
```
        0
        |
    9000|Server3
        |
    5000|Server2
        |
    1000|Server1
        |
        0
```

### Step 3: Place Keys on Ring

**Hash each key** and place it on the ring:

```python
def hash_key(key):
    return hash(key) % (2**32)

# Example
keys = ['user:1', 'user:2', 'user:3', 'user:4', 'user:5']
key_positions = {
    'user:1': hash('user:1') % (2**32) = 2000
    'user:2': hash('user:2') % (2**32) = 6000
    'user:3': hash('user:3') % (2**32) = 9500
    'user:4': hash('user:4') % (2**32) = 500
    'user:5': hash('user:5') % (2**32) = 7500
}
```

### Step 4: Assign Keys to Servers

**Rule**: Each key belongs to the **first server encountered clockwise**.

**Algorithm**:
```python
def find_server(key, servers, ring_size=2**32):
    key_hash = hash(key) % ring_size
    
    # Sort servers by their hash position
    sorted_servers = sorted(servers, key=lambda s: hash(s) % ring_size)
    
    # Find first server with hash >= key_hash
    for server in sorted_servers:
        server_hash = hash(server) % ring_size
        if server_hash >= key_hash:
            return server
    
    # Wrap around: key is after all servers
    return sorted_servers[0]
```

**Visual Assignment**:
```
        0
        |
    9500|user:3 → Server3 (wrap around)
        |
    9000|Server3
        |
    7500|user:5 → Server3
        |
    6000|user:2 → Server2
        |
    5000|Server2
        |
    2000|user:1 → Server2
        |
    1000|Server1
        |
     500|user:4 → Server1
        |
        0
```

**Result**:
- `user:1` (2000) → Server2 (5000) ✅
- `user:2` (6000) → Server2 (5000) ✅
- `user:3` (9500) → Server3 (9000, wrap around) ✅
- `user:4` (500) → Server1 (1000) ✅
- `user:5` (7500) → Server3 (9000) ✅

## Key Concepts

### 1. Hash Ring (Circle)

**Definition**: A circular space representing all possible hash values.

**Size**: Typically 2^32 (4 billion) or 2^64 (18 quintillion)

**Properties**:
- **Circular**: After 2^32 - 1, wraps back to 0
- **Uniform**: Hash function distributes evenly
- **Deterministic**: Same input always produces same hash

### 2. Server Placement

**Method**: Hash server identifier (IP, hostname, or unique ID)

**Example**:
```python
server_hash = hash('192.168.1.100:6379') % (2**32)
```

**Important**: Server position is **fixed** (unless server ID changes)

### 3. Key Placement

**Method**: Hash key value

**Example**:
```python
key_hash = hash('user:12345') % (2**32)
```

**Important**: Key position is **fixed** (deterministic)

### 4. Clockwise Assignment

**Rule**: Find the first server encountered when moving clockwise from key position.

**Edge Case**: If key is after all servers, wrap around to first server.

**Visual**:
```
Key at position 9500:
    → Check Server1 (1000): 9500 < 1000? No (wrap around)
    → Check Server2 (5000): 9500 < 5000? No (wrap around)
    → Check Server3 (9000): 9500 >= 9000? Yes! ✅
    → Assign to Server3
```

### 5. Minimal Remapping

**Key Benefit**: When a server is added/removed, only keys in the **affected range** need remapping.

**Example - Adding Server4**:
```
Before: Server1 (1000), Server2 (5000), Server3 (9000)
After:  Server1 (1000), Server2 (5000), Server3 (9000), Server4 (3000)

Affected Range: Keys between Server1 (1000) and Server4 (3000)
- Only keys in range [1000, 3000) need remapping
- Keys outside this range remain unchanged
```

**Remapping Ratio**: ~1/n where n = number of servers (much better than 100%!)

## Algorithm Details

### Basic Algorithm

```python
import hashlib

class ConsistentHash:
    def __init__(self, ring_size=2**32):
        self.ring_size = ring_size
        self.servers = []
        self.server_positions = {}
    
    def _hash(self, value):
        """Hash function (MD5, SHA1, or CRC32)"""
        # Using MD5 and taking first 4 bytes as 32-bit integer
        hash_obj = hashlib.md5(str(value).encode())
        hash_hex = hash_obj.hexdigest()
        return int(hash_hex[:8], 16) % self.ring_size
    
    def add_server(self, server_id):
        """Add a server to the ring"""
        if server_id not in self.servers:
            self.servers.append(server_id)
            self.server_positions[server_id] = self._hash(server_id)
    
    def remove_server(self, server_id):
        """Remove a server from the ring"""
        if server_id in self.servers:
            self.servers.remove(server_id)
            del self.server_positions[server_id]
    
    def get_server(self, key):
        """Get the server for a given key"""
        if not self.servers:
            return None
        
        key_hash = self._hash(key)
        
        # Sort servers by position
        sorted_servers = sorted(
            self.servers,
            key=lambda s: self.server_positions[s]
        )
        
        # Find first server with position >= key_hash
        for server in sorted_servers:
            if self.server_positions[server] >= key_hash:
                return server
        
        # Wrap around: return first server
        return sorted_servers[0]
    
    def get_servers(self):
        """Get all servers"""
        return sorted(
            self.servers,
            key=lambda s: self.server_positions[s]
        )
```

### Usage Example

```python
# Create consistent hash ring
ch = ConsistentHash()

# Add servers
ch.add_server('Server1')
ch.add_server('Server2')
ch.add_server('Server3')

# Assign keys

print(ch.get_server('user:2'))  # Server2

print(ch.get_server('user:5'))  # Server3
```

### Optimized Algorithm (Binary Search)

**Problem**: Linear search through servers is O(n)

**Solution**: Use binary search for O(log n) lookup

```python
import bisect

class OptimizedConsistentHash:
    def __init__(self, ring_size=2**32):
        self.ring_size = ring_size
        self.servers = []  # List of server IDs
        self.server_positions = {}  # server_id -> position
        self.sorted_positions = []  # Sorted list of positions
    
    def _hash(self, value):
        hash_obj = hashlib.md5(str(value).encode())
        hash_hex = hash_obj.hexdigest()
        return int(hash_hex[:8], 16) % self.ring_size
    
    def add_server(self, server_id):
        if server_id not in self.servers:
            self.servers.append(server_id)
            position = self._hash(server_id)
            self.server_positions[server_id] = position
            
            # Insert position in sorted order
            bisect.insort(self.sorted_positions, position)
    
    def remove_server(self, server_id):
        if server_id in self.servers:
            self.servers.remove(server_id)
            position = self.server_positions[server_id]
            del self.server_positions[server_id]
            
            # Remove position from sorted list
            self.sorted_positions.remove(position)
    
    def get_server(self, key):
        if not self.servers:
            return None
        
        key_hash = self._hash(key)
        
        # Binary search for first position >= key_hash
        index = bisect.bisect_left(self.sorted_positions, key_hash)
        
        if index < len(self.sorted_positions):
            # Found server at or after key position
            position = self.sorted_positions[index]
            # Find server with this position
            for server_id, pos in self.server_positions.items():
                if pos == position:
                    return server_id
        
        # Wrap around: return server with smallest position
        position = self.sorted_positions[0]
        for server_id, pos in self.server_positions.items():
            if pos == position:
                return server_id
```

**Time Complexity**:
- **Add server**: O(log n) - binary search insertion
- **Remove server**: O(n) - linear search for removal
- **Get server**: O(log n) - binary search lookup

## Virtual Nodes (Vnodes)

### The Problem: Uneven Distribution

**Issue**: With few servers, keys may not distribute evenly.

**Example**:
```
Server1: 1000
Server2: 5000
Server3: 9000

Keys distribution:
- Range [0, 1000): 0 keys
- Range [1000, 5000): 4000 possible positions
- Range [5000, 9000): 4000 possible positions
- Range [9000, 2^32): 2^32 - 9000 possible positions

Result: Server3 gets most keys!
```

### Solution: Virtual Nodes

**Concept**: Each physical server is represented by **multiple virtual nodes** on the ring.

**Benefits**:
1. **Better distribution**: More points on ring = more even distribution
2. **Load balancing**: Keys spread across virtual nodes
3. **Fault tolerance**: When server fails, load distributed across remaining servers

### Implementation

```python
class ConsistentHashWithVnodes:
    def __init__(self, ring_size=2**32, vnodes_per_server=150):
        self.ring_size = ring_size
        self.vnodes_per_server = vnodes_per_server
        self.servers = []
        self.vnode_to_server = {}  # vnode_hash -> server_id
        self.sorted_vnodes = []  # Sorted list of vnode positions
    
    def _hash(self, value):
        hash_obj = hashlib.md5(str(value).encode())
        hash_hex = hash_obj.hexdigest()
        return int(hash_hex[:8], 16) % self.ring_size
    
    def add_server(self, server_id):
        if server_id not in self.servers:
            self.servers.append(server_id)
            
            # Create virtual nodes
            for i in range(self.vnodes_per_server):
                vnode_id = f"{server_id}#{i}"
                vnode_hash = self._hash(vnode_id)
                self.vnode_to_server[vnode_hash] = server_id
                bisect.insort(self.sorted_vnodes, vnode_hash)
    
    def remove_server(self, server_id):
        if server_id in self.servers:
            self.servers.remove(server_id)
            
            # Remove all virtual nodes
            vnodes_to_remove = [
                vnode_hash for vnode_hash, srv_id in self.vnode_to_server.items()
                if srv_id == server_id
            ]
            
            for vnode_hash in vnodes_to_remove:
                del self.vnode_to_server[vnode_hash]
                self.sorted_vnodes.remove(vnode_hash)
    
    def get_server(self, key):
        if not self.servers:
            return None
        
        key_hash = self._hash(key)
        
        # Find first vnode >= key_hash
        index = bisect.bisect_left(self.sorted_vnodes, key_hash)
        
        if index < len(self.sorted_vnodes):
            vnode_hash = self.sorted_vnodes[index]
            return self.vnode_to_server[vnode_hash]
        
        # Wrap around
        vnode_hash = self.sorted_vnodes[0]
        return self.vnode_to_server[vnode_hash]
```

## Advantages and Benefits

### 1. Minimal Remapping

**Traditional Hashing**: Adding/removing server remaps **100% of keys**

**Consistent Hashing**: Adding/removing server remaps **~1/n of keys** (where n = number of servers)

**Example**:
- 3 servers: Remap ~33% of keys
- 10 servers: Remap ~10% of keys
- 100 servers: Remap ~1% of keys

### 2. Horizontal Scalability

**Easy to Scale**: Add servers without massive data movement

**Example**:
```
Initial: 3 servers, 1M keys
Add 1 server: Only ~250K keys need to move (25%)
Total data movement: Minimal
```

### 3. Load Balancing

**Even Distribution**: Keys distributed evenly across servers (with virtual nodes)

**Example**:
```
3 servers, 1M keys:
- Server1: ~333K keys
- Server2: ~333K keys
- Server3: ~334K keys
```

### 4. Fault Tolerance

**Server Failure**: Only keys on failed server need remapping

**Example**:
```
3 servers, Server2 fails:
- Server1: Unchanged
- Server2: Keys remapped to Server3
- Server3: Receives Server2's keys
```

### 5. No Central Coordinator

**Decentralized**: Each client can compute server assignment independently

**Benefits**:
- No single point of failure
- No coordination overhead
- Scalable to many clients

### 6. Deterministic

**Same Input = Same Output**: Same key always maps to same server (until server changes)

**Benefits**:
- Predictable behavior
- Easy to debug
- Cache-friendly

## Limitations and Challenges

### 1. Uneven Distribution (Without Virtual Nodes)

**Problem**: With few servers, distribution can be uneven

**Solution**: Use virtual nodes

### 2. Hot Spots

**Problem**: Popular keys may cluster on one server

**Example**:
```
Popular user IDs hash to similar values
→ All assigned to same server
→ Server overloaded
```

**Solutions**:
- Use better hash function
- Add more virtual nodes
- Use consistent hashing with bounded loads

### 3. Key Skew

**Problem**: Some keys accessed more frequently than others

**Example**:
```
Key 'user:1' accessed 1M times/day
Key 'user:2' accessed 10 times/day
Both on same server → Uneven load
```

**Solutions**:
- Replication
- Caching
- Load-aware consistent hashing

### 4. Server Capacity Differences

**Problem**: Servers have different capacities (CPU, memory, disk)

**Example**:
```
Server1: 100GB capacity
Server2: 500GB capacity
Consistent hashing assigns equal keys → Server1 overloaded
```

**Solutions**:
- Weighted consistent hashing
- More virtual nodes for larger servers
- Capacity-aware assignment

### 5. Ring Size Limitations

**Problem**: Hash collisions possible (though rare)

**Example**:
```
2^32 possible positions
Many servers/keys → Collision probability increases
```

**Solutions**:
- Use larger ring (2^64)
- Handle collisions (chaining, open addressing)

### 6. Rehashing Overhead

**Problem**: Still need to move some data when servers change

**Example**:
```
Add 1 server to 10-server cluster
→ ~10% of keys need remapping
→ Network traffic, cache invalidation
```

**Mitigation**:
- Batch operations
- Gradual migration
- Replication for availability

```python
import hashlib

class ConsistentHashRing:

    """
    
    def __init__(self, ring_size: int = 2**32, vnodes_per_server: int = 150):
        """
        Initialize consistent hash ring.
        
        Args:
            ring_size: Size of hash ring (default: 2^32)
            vnodes_per_server: Number of virtual nodes per physical server
        """
        self.ring_size = ring_size
        self.vnodes_per_server = vnodes_per_server
        self.servers: List[str] = []
        self.vnode_to_server: Dict[int, str] = {}  # vnode_hash -> server_id
        self.sorted_vnodes: List[int] = []  # Sorted vnode positions
    
    def _hash(self, value: str) -> int:
        """
        Hash function using MD5.
        
        Args:
            value: String to hash
            
        Returns:
            Hash value in range [0, ring_size)
        """
        hash_obj = hashlib.md5(str(value).encode('utf-8'))
        hash_hex = hash_obj.hexdigest()
        # Use first 8 hex digits as 32-bit integer
        return int(hash_hex[:8], 16) % self.ring_size
    
    def add_server(self, server_id: str) -> None:
        """
        Add a server to the ring.
        
        Args:
            server_id: Unique server identifier
        """
        if server_id in self.servers:
            return  # Server already exists
        
        self.servers.append(server_id)
        
        # Create virtual nodes
        for i in range(self.vnodes_per_server):
            vnode_id = f"{server_id}#{i}"
            vnode_hash = self._hash(vnode_id)
            
            # Handle collisions (unlikely but possible)
            while vnode_hash in self.vnode_to_server:
                vnode_id = f"{server_id}#{i}#{hash(str(vnode_hash))}"
                vnode_hash = self._hash(vnode_id)
            
            self.vnode_to_server[vnode_hash] = server_id
            bisect.insort(self.sorted_vnodes, vnode_hash)
    
    def remove_server(self, server_id: str) -> None:
        """
        Remove a server from the ring.
        
        Args:
            server_id: Server identifier to remove
        """
        if server_id not in self.servers:
            return  # Server doesn't exist
        
        self.servers.remove(server_id)
        
        # Remove all virtual nodes for this server
        vnodes_to_remove = [
            vnode_hash
            for vnode_hash, srv_id in self.vnode_to_server.items()
            if srv_id == server_id
        ]
        
        for vnode_hash in vnodes_to_remove:
            del self.vnode_to_server[vnode_hash]
            self.sorted_vnodes.remove(vnode_hash)
    
    def get_server(self, key: str) -> Optional[str]:
        """
        Get the server for a given key.
        
        Args:
            key: Key to look up
            
        Returns:
            Server identifier, or None if no servers
        """
        if not self.servers:
            return None
        
        key_hash = self._hash(key)
        
        # Binary search for first vnode >= key_hash
        index = bisect.bisect_left(self.sorted_vnodes, key_hash)
        
        if index < len(self.sorted_vnodes):
            # Found vnode at or after key position
            vnode_hash = self.sorted_vnodes[index]
            return self.vnode_to_server[vnode_hash]
        
        # Wrap around: return server with smallest vnode
        vnode_hash = self.sorted_vnodes[0]
        return self.vnode_to_server[vnode_hash]
    
    def get_servers(self) -> List[str]:
        """
        Get all servers in sorted order by position.
        
        Returns:
            List of server identifiers
        """
        return sorted(
            self.servers,
            key=lambda s: min(
                vnode_hash
                for vnode_hash, srv_id in self.vnode_to_server.items()
                if srv_id == s
            )
        )
    
    def get_key_distribution(self, keys: List[str]) -> Dict[str, int]:
        """
        Get distribution of keys across servers.
        
        Args:
            keys: List of keys to analyze
            
        Returns:
            Dictionary mapping server_id to number of keys
        """
        distribution = {server: 0 for server in self.servers}
        
        for key in keys:
            server = self.get_server(key)
            if server:
                distribution[server] += 1
        
        return distribution


# Usage Example
```

### Java: Basic Implementation

```java
import java.security.MessageDigest;
import java.security.NoSuchAlgorithmException;
import java.util.*;

public class ConsistentHashRing {
    private static final int RING_SIZE = Integer.MAX_VALUE; // 2^31 - 1
    private final int vnodesPerServer;
    private final List<String> servers;
    private final TreeMap<Long, String> vnodeToServer; // position -> server_id
    private MessageDigest md5;
    
    public ConsistentHashRing(int vnodesPerServer) {
        this.vnodesPerServer = vnodesPerServer;
        this.servers = new ArrayList<>();
        this.vnodeToServer = new TreeMap<>();
        
        try {
            this.md5 = MessageDigest.getInstance("MD5");
        } catch (NoSuchAlgorithmException e) {
            throw new RuntimeException("MD5 not available", e);
        }
    }
    
    private long hash(String value) {
        md5.reset();
        md5.update(value.getBytes());
        byte[] digest = md5.digest();
        
        // Use first 4 bytes as 32-bit integer
        long hash = 0;
        for (int i = 0; i < 4; i++) {
            hash = (hash << 8) | (digest[i] & 0xFF);
        }
        
        return Math.abs(hash) % RING_SIZE;
    }
    
    public void addServer(String serverId) {
        if (servers.contains(serverId)) {
            return;
        }
        
        servers.add(serverId);
        
        // Create virtual nodes
        for (int i = 0; i < vnodesPerServer; i++) {
            String vnodeId = serverId + "#" + i;
            long vnodeHash = hash(vnodeId);
            vnodeToServer.put(vnodeHash, serverId);
        }
    }
    
    public void removeServer(String serverId) {
        if (!servers.contains(serverId)) {
            return;
        }
        
        servers.remove(serverId);
        
        // Remove all virtual nodes
        vnodeToServer.entrySet().removeIf(entry -> entry.getValue().equals(serverId));
    }
    
    public String getServer(String key) {
        if (servers.isEmpty()) {
            return null;
        }
        
        long keyHash = hash(key);
        
        // Find first vnode >= keyHash
        Map.Entry<Long, String> entry = vnodeToServer.ceilingEntry(keyHash);
        
        if (entry != null) {
            return entry.getValue();
        }
        
        // Wrap around: return first vnode
        return vnodeToServer.firstEntry().getValue();
    }
    
    public List<String> getServers() {
        return new ArrayList<>(servers);
    }
}
```

### Go: Implementation

```go
package main

import (
    "crypto/md5"
    "fmt"
    "sort"
    "sync"
)

type ConsistentHashRing struct {
    ringSize        uint32
    vnodesPerServer int
    servers         []string
    vnodeToServer   map[uint32]string
    sortedVnodes    []uint32
    mutex           sync.RWMutex
}

func NewConsistentHashRing(vnodesPerServer int) *ConsistentHashRing {
    return &ConsistentHashRing{
        ringSize:        1 << 32, // 2^32
        vnodesPerServer: vnodesPerServer,
        servers:         make([]string, 0),
        vnodeToServer:   make(map[uint32]string),
        sortedVnodes:    make([]uint32, 0),
    }
}

func (r *ConsistentHashRing) hash(value string) uint32 {
    h := md5.Sum([]byte(value))
    // Use first 4 bytes as 32-bit integer
    return uint32(h[0])<<24 | uint32(h[1])<<16 | uint32(h[2])<<8 | uint32(h[3])
}

func (r *ConsistentHashRing) AddServer(serverID string) {
    r.mutex.Lock()
    defer r.mutex.Unlock()
    
    // Check if server already exists
    for _, s := range r.servers {
        if s == serverID {
            return
        }
    }
    
    r.servers = append(r.servers, serverID)
    
    // Create virtual nodes
    for i := 0; i < r.vnodesPerServer; i++ {
        vnodeID := fmt.Sprintf("%s#%d", serverID, i)
        vnodeHash := r.hash(vnodeID) % r.ringSize
        
        r.vnodeToServer[vnodeHash] = serverID
        r.sortedVnodes = append(r.sortedVnodes, vnodeHash)
    }
    
    // Sort vnodes
    sort.Slice(r.sortedVnodes, func(i, j int) bool {
        return r.sortedVnodes[i] < r.sortedVnodes[j]
    })
}

func (r *ConsistentHashRing) RemoveServer(serverID string) {
    r.mutex.Lock()
    defer r.mutex.Unlock()
    
    // Remove server from list
    for i, s := range r.servers {
        if s == serverID {
            r.servers = append(r.servers[:i], r.servers[i+1:]...)
            break
        }
    }
    
    // Remove virtual nodes
    newVnodes := make([]uint32, 0)
    for vnodeHash, srvID := range r.vnodeToServer {
        if srvID != serverID {
            newVnodes = append(newVnodes, vnodeHash)
        } else {
            delete(r.vnodeToServer, vnodeHash)
        }
    }
    r.sortedVnodes = newVnodes
    sort.Slice(r.sortedVnodes, func(i, j int) bool {
        return r.sortedVnodes[i] < r.sortedVnodes[j]
    })
}

func (r *ConsistentHashRing) GetServer(key string) string {
    r.mutex.RLock()
    defer r.mutex.RUnlock()
    
    if len(r.servers) == 0 {
        return ""
    }
    
    keyHash := r.hash(key) % r.ringSize
    
    // Binary search for first vnode >= keyHash
    idx := sort.Search(len(r.sortedVnodes), func(i int) bool {
        return r.sortedVnodes[i] >= keyHash
    })
    
    if idx < len(r.sortedVnodes) {
        vnodeHash := r.sortedVnodes[idx]
        return r.vnodeToServer[vnodeHash]
    }
    
    // Wrap around
    return r.vnodeToServer[r.sortedVnodes[0]]
}
```

## Comparison with Other Approaches

### Consistent Hashing vs. Traditional Hashing

| Aspect | Traditional Hashing | Consistent Hashing |
|--------|-------------------|-------------------|
| **Remapping on Add** | 100% of keys | ~1/n of keys |
| **Remapping on Remove** | 100% of keys | ~1/n of keys |
| **Distribution** | Even (with good hash) | Even (with vnodes) |
| **Complexity** | O(1) lookup | O(log n) lookup |
| **Scalability** | Poor | Excellent |
| **Use Case** | Static server count | Dynamic server count |

### Consistent Hashing vs. Range Partitioning

| Aspect | Range Partitioning | Consistent Hashing |
|--------|------------------|-------------------|
| **Key Assignment** | Based on key range | Based on hash |
| **Hot Spots** | Possible (popular ranges) | Less likely |
| **Load Balancing** | Manual tuning | Automatic |
| **Remapping** | Entire ranges | Minimal keys |
| **Complexity** | O(log n) lookup | O(log n) lookup |

### Consistent Hashing vs. Directory-Based

| Aspect | Directory-Based | Consistent Hashing |
|--------|----------------|-------------------|
| **Centralization** | Central directory | Decentralized |
| **Single Point of Failure** | Yes | No |
| **Lookup Speed** | O(1) | O(log n) |
| **Scalability** | Limited | Excellent |
| **Consistency** | Strong | Eventual |

## Best Practices

### 1. Use Virtual Nodes

**Always use virtual nodes** for better distribution:

```python
# Good: 150-200 virtual nodes per server
ring = ConsistentHashRing(vnodes_per_server=150)

# Bad: No virtual nodes (uneven distribution)
ring = ConsistentHashRing(vnodes_per_server=1)
```

### 2. Choose Good Hash Function

**Use cryptographic hash** (MD5, SHA1) for even distribution:

```python
# Good: MD5 or SHA1
hash = hashlib.md5(value).hexdigest()

# Bad: Simple hash (may have patterns)
hash = sum(ord(c) for c in value) % ring_size
```

### 3. Handle Server Failures Gracefully

**Implement retry logic** and failover:

```python
def get_with_failover(key, ring):
    primary_server = ring.get_server(key)
    try:
        return get_from_server(primary_server, key)
    except ServerError:
        # Try next server in ring
        servers = ring.get_servers()
        idx = servers.index(primary_server)
        next_server = servers[(idx + 1) % len(servers)]
        return get_from_server(next_server, key)
```

### 4. Monitor Key Distribution

**Track distribution** to detect imbalances:

```python
def monitor_distribution(ring, sample_keys):
    distribution = ring.get_key_distribution(sample_keys)
    
    # Check for imbalance
    counts = list(distribution.values())
    avg = sum(counts) / len(counts)
    max_deviation = max(abs(c - avg) for c in counts)
    
    if max_deviation > avg * 0.2:  # 20% deviation
        print("Warning: Uneven distribution detected")
```

### 5. Use Consistent Hash for Session Affinity

**Route same client** to same server:

```python
def route_request(request, ring):
    # Use client IP or session ID
    client_id = request.remote_addr
    server = ring.get_server(client_id)
    return forward_to(server, request)
```

### 6. Implement Replication

**Replicate data** for fault tolerance:

```python
def get_replicas(key, ring, replication_factor=3):
    primary = ring.get_server(key)
    servers = ring.get_servers()
    idx = servers.index(primary)
    
    replicas = []
    for i in range(replication_factor):
        replica = servers[(idx + i) % len(servers)]
        replicas.append(replica)
    
    return replicas
```

### 7. Handle Ring Size Carefully

**Use appropriate ring size**:

```python
# Good: 2^32 or 2^64
ring_size = 2**32  # 4 billion positions

# Bad: Too small (collisions)
ring_size = 1000  # Only 1000 positions
```

### 8. Cache Server Assignments

**Cache lookups** for performance:

```python
class CachedConsistentHash:
    def __init__(self, ring):
        self.ring = ring
        self.cache = {}
    
    def get_server(self, key):
        if key in self.cache:
            return self.cache[key]
        
        server = self.ring.get_server(key)
        self.cache[key] = server
        return server
    
    def invalidate_cache(self):
        self.cache.clear()
```

## Summary

### Key Takeaways

1. **Consistent Hashing** minimizes remapping when servers are added/removed
2. **Hash Ring** maps both keys and servers to circular space
3. **Virtual Nodes** improve distribution and load balancing
4. **Remapping Ratio**: ~1/n (much better than 100%)
5. **Use Cases**: Caching, load balancing, distributed databases, CDN
6. **Best Practices**: Use virtual nodes, good hash function, replication

### When to Use Consistent Hashing

✅ **Use When**:
- Dynamic server count (adding/removing servers)
- Need minimal remapping
- Distributed caching or storage
- Load balancing with session affinity
- Horizontal scaling

❌ **Don't Use When**:
- Static server count (traditional hashing sufficient)
- Need strict ordering
- Need range queries
- Small number of servers (< 3)

### Performance Characteristics

| Operation | Time Complexity | Space Complexity |
|-----------|----------------|------------------|
| **Add Server** | O(v × log n) | O(v) |
| **Remove Server** | O(v × log n) | O(v) |
| **Get Server** | O(log n) | O(1) |
| **Lookup** | O(log n) | O(1) |

Where:
- `n` = number of virtual nodes
- `v` = virtual nodes per server
