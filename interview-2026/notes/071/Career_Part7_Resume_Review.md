# Engineering Resume Review - Ex-Google Recruiter

## Overview

This guide provides insights from a top ex-Google recruiter on engineering resume reviews. It covers what recruiters look for, common mistakes, and how to make your resume stand out.

## Recruiter's Perspective

```
┌─────────────────────────────────────────────────────────┐
│         Recruiter Review Process                       │
└─────────────────────────────────────────────────────────┘

Initial Scan (6 seconds):
├─► Format and readability
├─► Key skills match
├─► Years of experience
└─► Education background

Detailed Review (30 seconds):
├─► Quantifiable achievements
├─► Technical depth
├─► Impact and results
└─► Career progression

Decision:
├─► Pass to hiring manager
├─► Reject
└─► Maybe (needs more review)
```

## What Recruiters Look For

### 1. Technical Skills Match

```
┌─────────────────────────────────────────────────────────┐
│         Skills Assessment                               │
└─────────────────────────────────────────────────────────┘

Key Factors:
├─► Relevant technologies
├─► Depth of experience
├─► Recent experience
├─► Certifications (if relevant)
└─► Open source contributions

Red Flags:
├─► Keyword stuffing
├─► Outdated technologies only
├─► No depth in any area
└─► Skills don't match experience
```

### 2. Quantifiable Impact

```
┌─────────────────────────────────────────────────────────┐
│         Impact Examples                                 │
└─────────────────────────────────────────────────────────┘

Strong Examples:
✅ "Reduced API latency by 40%, improving user experience"

❌ "Led a team"
❌ "Reduced costs"

```
┌─────────────────────────────────────────────────────────┐

Positive Signs:
├─► Increasing responsibility

└─► Broader impact

Concerning Signs:

## Common Resume Mistakes

### Mistake 1: No Metrics

```
BAD:
"Improved system performance"
"Led development team"
"Worked on scaling infrastructure"

GOOD:
"Improved system performance by 50%, reducing response time from
 500ms to 250ms, serving 10M+ daily active users"
"Led team of 6 engineers to deliver microservices architecture,
 reducing deployment time by 80% and improving system reliability"
"Scaled infrastructure to handle 5x traffic growth while reducing
 costs by 30% through optimization"
```

### Mistake 2: Weak Action Verbs

```
WEAK:
"Worked on", "Helped with", "Was involved in", "Assisted"

STRONG:
"Architected", "Led", "Designed", "Implemented", "Optimized",
"Built", "Scaled", "Transformed", "Mentored", "Delivered"
```

### Mistake 3: Too Generic

```
GENERIC:
"Software Engineer responsible for developing features"

SPECIFIC:
"Senior Software Engineer building distributed systems using
 Java and Spring Boot, serving 5M+ users with 99.9% uptime"
```

### Mistake 4: Poor Formatting

```
┌─────────────────────────────────────────────────────────┐
│         Formatting Issues                              │
└─────────────────────────────────────────────────────────┘

Problems:
├─► Inconsistent spacing
├─► Multiple fonts
├─► Poor alignment
├─► Too dense
├─► Hard to scan
└─► ATS-unfriendly

Solutions:
├─► Consistent formatting
├─► Clear sections
├─► Adequate white space
├─► Standard fonts
├─► Bullet points
└─► ATS-friendly format
```

### Mistake 5: Irrelevant Information

```
┌─────────────────────────────────────────────────────────┐
│         What to Include/Exclude                        │
└─────────────────────────────────────────────────────────┘

Include:
✅ Relevant work experience
✅ Technical projects
✅ Quantifiable achievements
✅ Education (if recent/relevant)
✅ Key skills

Exclude:
❌ Unrelated work experience
❌ Personal information (age, photo)
❌ Hobbies (unless relevant)
❌ References
❌ "References available upon request"
```

## Recruiter's Top Tips

### Tip 1: Lead with Impact

```
Structure each bullet:
[Action Verb] + [What] + [How] + [Impact/Result]

Example:
"Architected microservices platform using Java and Spring Boot,
 reducing latency by 45% and supporting 5M+ daily active users"
```

### Tip 2: Use Numbers

```
Always include:
├─► Performance improvements (%)
├─► Scale (users, requests, data)
├─► Cost savings ($)
├─► Time saved (hours, days)
├─► Team size (#)
└─► Revenue impact ($)
```

### Tip 3: Tailor to Job

```
For each application:
1. Read job description
2. Identify key requirements
3. Match your experience
4. Use same keywords
5. Highlight relevant projects
6. Adjust order
```

### Tip 4: Show Growth

```
Demonstrate progression:
├─► Increasing responsibility
├─► Larger impact
├─► More complex projects
├─► Leadership experience
└─► Technical depth
```

## Red Flags for Recruiters

```
┌─────────────────────────────────────────────────────────┐
│         Resume Red Flags                               │
└─────────────────────────────────────────────────────────┘

1. ❌ No quantifiable metrics
2. ❌ Spelling/grammar errors
3. ❌ Gaps in employment (unexplained)
4. ❌ Frequent job changes (<1 year)
5. ❌ Outdated technologies only
6. ❌ Skills don't match experience
7. ❌ Too long or too short
8. ❌ Poor formatting
9. ❌ Generic descriptions
10. ❌ No clear career progression
```

## Key Takeaways

### What Makes a Resume Stand Out:

1. ✅ **Quantifiable achievements** - Numbers in every bullet
2. ✅ **Strong action verbs** - Show impact, not just tasks
3. ✅ **Technical depth** - Demonstrate expertise
4. ✅ **Clear progression** - Show career growth
5. ✅ **Tailored content** - Match job requirements
6. ✅ **Professional format** - Clean and ATS-friendly
7. ✅ **Relevant projects** - Show practical experience

### Most Common Issues:

- No metrics (80% of resumes)
- Weak action verbs (70% of resumes)
- Generic descriptions (60% of resumes)
- Poor formatting (50% of resumes)
- Not tailored (40% of resumes)

## Summary

**Recruiter's Perspective:**
- First impression matters (6-second scan)
- Quantifiable impact is critical
- Technical depth must be evident
- Career progression should be clear
- Formatting affects readability

**Key Success Factors:**
1. Add metrics to every bullet point
2. Use strong action verbs
3. Show technical depth
4. Demonstrate impact
5. Tailor to each application
6. Maintain professional format
7. Proofread carefully

**Remember**: Recruiters review hundreds of resumes. Make yours easy to scan, clearly show your value, and stand out with quantifiable achievements!
