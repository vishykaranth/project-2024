# Mapping PoEAA → Modern Spring / Quarkus / Micronaut (2026)

> **Mental model to say in interviews**
> “PoEAA patterns didn’t disappear — frameworks absorbed them.”

---

## 🧠 1️⃣ Domain Logic Patterns → Modern Frameworks

### **Domain Model**

**PoEAA**

* Rich objects encapsulate behavior

**Modern**

* `@Entity` + business methods
* DDD aggregates (explicit or implicit)

**Spring**

```java
@Entity
class Order {
  void approve() { ... }
}
```

**Quarkus / Micronaut**

* Same model
* Encourages leaner entities

> **Interview line:**
> “Frameworks support Domain Model, but discipline keeps it clean.”

---

### **Transaction Script**

**PoEAA**

* Procedural logic per request

**Modern**

* `@Service` methods
* Lambda-style handlers
* Serverless functions

**Spring**

```java
@Service
@Transactional
public void processOrder(...) { ... }
```

> Use for CRUD-heavy services.

---

### **Service Layer**

**PoEAA**

* Defines application boundary

**Modern**

* Application services
* REST/GraphQL handlers delegate here

**Spring**

* `@Service`
* `@Transactional`

**Quarkus**

* `@ApplicationScoped`

**Micronaut**

* `@Singleton`

> **Key insight:**
> *Microservice ≠ Service Layer; it contains one.*

---

## 🗄️ 2️⃣ Data Access & ORM Patterns

### **Data Mapper**

**PoEAA**

* Separate domain & persistence

**Modern**

* JPA / Hibernate / Panache
* Repositories hide SQL

**Spring**

```java
interface OrderRepository extends JpaRepository<Order, Long>
```

**Quarkus**

* Panache Repository

**Micronaut**

* Micronaut Data

> ORM is Data Mapper on steroids.

---

### **Unit of Work**

**PoEAA**

* Track changes, commit once

**Modern**

* Persistence Context
* Transaction boundaries

**Spring**

* `@Transactional`

**Quarkus**

* `@Transactional` (Narayan)

**Micronaut**

* `@TransactionalAdvice`

> **Interview line:**
> “Unit of Work is the transaction.”

---

### **Identity Map**

**PoEAA**

* One object per DB row

**Modern**

* Hibernate Session
* Persistence Context

> Prevents duplicate loads automatically.

---

### **Lazy Load**

**PoEAA**

* Load on demand

**Modern**

* `FetchType.LAZY`
* Proxies

> Warning: N+1 problems didn’t die.

---

## 🔄 3️⃣ Concurrency & Correctness

### **Optimistic Offline Lock**

**PoEAA**

* Version-based conflict detection

**Modern**

```java
@Version
private Long version;
```

**Spring / Quarkus / Micronaut**

* Fully supported

> Essential for distributed systems.

---

### ❌ **Pessimistic Offline Lock**

**Modern status**

* Supported, rarely recommended

> “Locks don’t scale across services.”

---

## 🌐 4️⃣ Distribution & API Patterns

### **Remote Facade**

**PoEAA**

* Coarse-grained interface

**Modern**

* REST Controllers
* GraphQL Resolvers
* gRPC Services

**Spring**

```java
@RestController
class OrderController { ... }
```

> API boundary = Remote Facade.

---

### **DTO (Data Transfer Object)**

**PoEAA**

* Bundle data for transport

**Modern**

* Request/Response models
* Projections

**Spring**

```java
record OrderDTO(...) {}
```

**Micronaut**

* Compile-time DTO mapping

> **Interview line:**
> “DTOs protect domain models from leaking.”

---

### **Gateway**

**PoEAA**

* Encapsulate external systems

**Modern**

* `@FeignClient`
* WebClient
* HTTP/gRPC clients

> Mandatory for payments, auth, integrations.

---

## 🧱 5️⃣ Web Presentation Patterns

### **Front Controller**

**PoEAA**

* Single entry point

**Modern**

* Servlet Dispatcher
* Netty router

**Spring**

* `DispatcherServlet`

**Quarkus**

* Reactive routing

**Micronaut**

* Compile-time routing

> You don’t implement it; the framework is it.

---

### **MVC**

**PoEAA**

* Model, View, Controller

**Modern**

* Mostly backend MVC
* Frontend moved to SPAs

> MVC survived, but shrank.

---

## 🧳 6️⃣ Session State Patterns

### **Server Session State**

**PoEAA**

* In-memory session

**Modern**

* Avoided
* Externalized (Redis)

> Cloud killed sticky sessions.

---

### **Client Session State**

**PoEAA**

* State on client

**Modern**

* JWTs
* OAuth tokens

> Security > convenience.

---

## 🧩 7️⃣ Base Patterns → Framework Internals

| PoEAA Pattern       | Modern Equivalent                    |
| ------------------- | ------------------------------------ |

| Layer Supertype     | Base classes / aspects               |
| Plugin              | Spring starters / Quarkus extensions |
| Separated Interface | Interfaces + DI                      |
| Service Stub        | Mocks, WireMock, TestContainers      |
