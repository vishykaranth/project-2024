# 2️⃣ DEFINE THE TRANSACTION BOUNDARY (NON-NEGOTIABLE)

### LLD decision

> “Correctness lives inside transactions.”

@Service
public class OrderService {

### Why this is production-ready

* No nested transactions

🔑 **Kill shot embedded**

> *Correctness lives inside transactions; scale lives outside them.*

# 3️⃣ DOMAIN MODEL = INVARIANTS ONLY

### LLD decision

> “Only invariants live in the domain model.”

### Production code

```java
@Entity
public class Order {

    @Id
    private UUID id;

    @Enumerated(EnumType.STRING)
    private OrderStatus status;

    private BigDecimal total;

    public void confirm() {
        if (status != OrderStatus.CREATED) {
            throw new IllegalStateException("Invalid state transition");
        }
        this.status = OrderStatus.CONFIRMED;
    }
}
```

### What’s *not* here (important)

* No database calls
* No orchestration
* No external services

### Why this is production-ready

* Invariants are enforced centrally
* Easy to test
* Easy to reason about under concurrency

🔑 **Kill shot embedded**

> *Rich models protect invariants, not workflows.*

# 4️⃣ ORCHESTRATION = TRANSACTION SCRIPT

### LLD decision

> “Orchestration stays procedural.”

### Production code

```java
@Transactional
public OrderResult placeOrder(PlaceOrderCommand cmd) {

    idempotencyService.ensureNotProcessed(cmd.idempotencyKey());

    Inventory inventory = inventoryRepository.findBySku(cmd.sku());
    inventory.reserve(cmd.quantity());

    Order order = new Order(cmd.orderId(), cmd.total());
    order.confirm();

    orderRepository.save(order);

    eventPublisher.publishOrderPlaced(order.getId());

    return OrderResult.success(order.getId());
}
```

### Why this is production-ready

* Linear, readable flow
* Easy to debug at 3am
* Explicit sequence of side effects

🔑 **Kill shot embedded**

> *Transaction Scripts scale orchestration better than objects.*

# 5️⃣ CONCURRENCY CONTROL (OPTIMISTIC LOCKING)

### LLD decision

> “Protect shared state, redesign if contention spikes.”

### Production code

```java
@Entity
public class Inventory {

    @Id
    private String sku;

    @Version
    private long version;

    private int available;

    public void reserve(int qty) {
        if (available < qty) {
            throw new OutOfStockException();
        }
        available -= qty;
    }
}
```

### Why this is production-ready

* Prevents overselling
* Detects contention early
* Forces redesign instead of silent corruption

🔑 **Kill shot embedded**

> *If optimistic locking thrashes, the data model is wrong.*

# 6️⃣ IDEMPOTENCY (MANDATORY IN PROD)

### LLD decision

> “Retries are inevitable.”

### Production code

```java
@Service
public class IdempotencyService {

    public void ensureNotProcessed(String key) {
        if (!store.putIfAbsent(key)) {
            throw new DuplicateRequestException();
        }
    }
}
```

### Why this is production-ready

* Safe retries
* Safe client timeouts
* Safe message redelivery

🔑 **Kill shot embedded**

> *Retries without idempotency create duplicates.*

# 7️⃣ ASYNC FAILURE HANDLING (OUTSIDE TRANSACTION)

### LLD decision

> “Do not couple reliability of side effects to core flow.”

### Production code

```java
@Component
public class OrderEventPublisher {

    @TransactionalEventListener(phase = AFTER_COMMIT)
    public void publish(OrderPlacedEvent event) {
        kafkaTemplate.send("orders", event);
    }
}
```

### Why this is production-ready

* No ghost events
* No partial commits
* Clean separation of concerns

🔑 **Kill shot embedded**

> *Messaging buys time, not correctness.*

# 8️⃣ FAILURE HANDLING STRATEGY (VISIBLE IN CODE)

try {

🔑 **Kill shot embedded**
