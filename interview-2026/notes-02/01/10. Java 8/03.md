# 1️⃣ Side Effects in `forEach` (DATA CORRUPTION)

### ❌ Bug

```java
List<Order> processed = new ArrayList<>();

orders.parallelStream()
      .forEach(o -> processed.add(o));
```

### 💥 What goes wrong

* `ArrayList` is not thread-safe
* Parallel stream causes lost writes or corruption

### ✅ Fix

```java
List<Order> processed =
    orders.parallelStream().toList();
```

or

```java
Collections.synchronizedList(new ArrayList<>());
```

### Kill shot

> **Streams assume immutability; side effects break the contract.**

# 4️⃣ `peek()` Used for Logic (HEISENBUG)

### ❌ Bug

```java
orders.stream()
      .peek(o -> o.setStatus(PROCESSED))
      .filter(o -> o.amount > 1000)
      .toList();
```

### 💥 What goes wrong

* `peek` may not run if terminal op changes
* Order-dependent side effects

### ✅ Fix

```java
orders.stream()
      .filter(o -> o.amount > 1000)
      .map(o -> {
          o.setStatus(PROCESSED);
          return o;
      })
      .toList();
```

### Kill shot

> **`peek` is for debugging, not behavior.**

# 5️⃣ Stream Reuse in Refactor (RUNTIME FAILURE)

### ❌ Bug

```java
Stream<Order> s = orders.stream();

if (debug) s.forEach(log::info);

List<Order> result = s.filter(o -> o.amount > 100).toList();
```

### 💥 What goes wrong

`IllegalStateException: stream has already been operated upon`

### ✅ Fix

```java
Supplier<Stream<Order>> streamSupplier = orders::stream;
```

### Kill shot

> **Streams are consumables, not collections.**
